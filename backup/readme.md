# Задание 3
```
3.1. С помощью официальной документации приведите пример команды инкрементного резервного копирования базы данных MySQL.
3.1.* В каких случаях использование реплики будет давать преимущество по сравнению с обычным резервным копированием?
```
```
3.1
mysqlbackup --incremental --incremental-base=history:last_backup --backup-dir=/path/to/backup/dir --user=username --password=password --host=hostname backup
  --incremental: Режим инкрементного резервного копирования.
  --incremental-base=history:last_backup: База для инкрементного резервного копирования, основанная на последнем полном резервном копировании.
  --backup-dir=/path/to/backup/dir: Каталог для сохранения резервной копии.
  --user=username: Имя пользователя для подключения.
  --password=password: Пароль пользователя для подключения.
  --host=hostname: Имя хоста, где расположена база данных.

3.1*.
Репликация обеспечивает непрерывную доступность данных. Если основной сервер выходит из строя, реплика может быть быстро переключена в активный режим.
Чтение данных может быть распределено между основным сервером и репликами, что снижает нагрузку на основной сервер и улучшает производительность системы.
Репликация обеспечивает непрерывное обновление данных, что позволяет минимизировать время простоя при сбоях.
Реплики могут использоваться для быстрого восстановления данных без необходимости длительного процесса восстановления из резервной копии.
```
